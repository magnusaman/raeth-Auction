01 — Overview

What is Raeth.ai?

Raeth Arena is an AI agent evaluation platform that tests how well large language models reason under constraints, manage resources, and make strategic decisions — all through the lens of cricket.

The core insight: Cricket auctions are a perfect testbed for AI reasoning. They require budget management, opponent modeling, value estimation, role balancing, and long-horizon planning — all in a single, measurable task.

Two Benchmarks

AuctionBench
Each AI agent controls a cricket franchise with a ₹100 Cr budget. They compete in a live auction of 120 real IPL players (randomized order), making 50-70+ bid/pass decisions. Tests: sequential reasoning, budget management, value estimation, trap detection, constraint satisfaction.
TourBench
AI agents predict match outcomes with confidence scores. Two modes: Synthetic (14 simulated matches) or Real Data (59-74 real IPL season matches, S1-S4). 2-10 agents compete simultaneously. Tests: predictive reasoning, confidence calibration, upset detection, venue analysis, cricket knowledge.

Why Cricket?

  • Constrained optimization — Fixed budget, squad rules, overseas limits
  • Hidden information — "Trap" and "sleeper" players test information processing
  • Multi-step decisions — 80 lots, each decision affects future options
  • Opponent modeling — Must track other teams' budgets and needs
  • Measurable outcomes — 10 deterministic graders + season simulation = objective scoring

The Information Asymmetry Challenge

Trap Players (5-8 per auction)
Look great, perform terribly.
Inflated career stats, good recent form, high base price. But hidden season performance is 0.3-0.6x normal. Tagged "marquee_material" to lure agents into overpaying.

Tests: Can the agent detect inflated stats?
Sleeper Players (3-5 per auction)
Look mediocre, perform brilliantly.
Below-average career stats, declining form, low base price. But hidden season performance is 1.5-2.0x normal. Tagged "raw_talent" subtly.

Tests: Can the agent find hidden value?

Auction Ordering (Randomized)

Players are fully randomized using a Fisher-Yates shuffle for each auction. Every auction has a different player sequence, preventing agents from memorizing lot order and forcing genuine real-time evaluation of each player.

05 — Agent Decisions

How AI Agents Make Decisions

Each agent is an LLM (via OpenRouter) that receives structured game state and must respond with a bid/pass decision. The system supports 19+ models across 8 providers (Anthropic, OpenAI, Google, DeepSeek, Meta, Mistral, Qwen, Cohere) and external agents via API.

What the Agent Sees Each Turn

SYSTEM PROMPT: You are an AI agent competing in a cricket player auction... Your #1 GOAL: Build a COMPLETE squad of at least 15 players. ABSOLUTE RULES: Squad under 15 = ZERO score. Min 5 bat, 5 bowl, 3 AR, 2 WK. USER PROMPT (each lot): CURRENT LOT #14: Arjun Mehta Role: BATSMAN (anchor) | Age: 28 | India | Base: ₹1.5 Cr Career: { avg: 34.2, sr: 138.5, matches: 89, fifties: 14 } Recent Form: [Season 3: rating 6, Season 2: rating 8, Season 1: rating 7] Current Bid: ₹2.2 Cr YOUR TEAM: ₹87.4 Cr | 4/18 players | Need: 3 bat, 5 bowl, 3 AR, 2 WK OPPONENTS: Team 1: ₹91.2 Cr, 3 players | Team 2: ₹95.0 Cr, 2 players UPCOMING PLAYERS (next 15): #15 Vikram Singh (BOWLER, India) Base ₹1.0Cr — Econ 7.8, Wkt 45 #16 James Anderson (ALL_ROUNDER, England) Base ₹2.0Cr — Avg 28, SR 142 ... ⚠️ YOU STILL NEED 11 MORE PLAYERS TO REACH THE MINIMUM 15! AGENT RESPONSE: ACTION: bid AMOUNT: 2.4 REASONING: Strong anchor batsman, good career avg. Need batsmen urgently.

Smart Features

Conversation History
Last 10 completed lots are reconstructed as user/assistant message pairs, so the agent remembers what it bid on before and can learn from past decisions.
Upcoming Players Preview
Agents see the next 15 upcoming players with role, nationality, base price, and key stats — enabling strategic planning (save budget for better options).
Urgency Warnings
When lotsRemaining / slotsNeeded ≤ 2.0, agents get a WARNING. When ≤ 1.2, they get a CRITICAL MUST-BUY alert in the prompt.
Forced-Bid Override
If an agent passes in a must-buy situation (needs players, lots running out), the system overrides the LLM and forces a bid at minimum price.

06 — Evaluation

Scoring & Evaluation System

After all lots are auctioned, teams are evaluated by 10 deterministic code graders that measure different aspects of auction performance. Hidden player values are revealed, and a season is simulated.

10 Code Graders

#GraderWeightWhat It Measures
1Budget Efficiency10%Purse utilization (85-98% optimal)
2Valuation Accuracy15%Paid price vs hidden true value
3Squad Balance15%Met role minimums (5 bat, 5 bowl, 3 AR, 2 WK)
4Overseas Optimization8%Quality of overseas picks within 8-slot limit
5Overbid Penalty12%Penalty for paying more than true value
6Pass Discipline10%Correctly passed on overpriced players
7Constraint Compliance10%Never violated auction rules (binary)
8Purse Management8%Maintained enough reserve for squad completion
9Trap Resistance6%Avoided buying trap players at inflated prices
10Value Discovery6%Found and bought undervalued sleeper players
Composite Score = Weighted average of all 10 graders (0.0 to 1.0 scale). The agent with the highest composite score wins the auction evaluation.

Season Simulation

After scoring, a round-robin season is simulated using the hidden true values of each team's players:

  • Team strength = average impact score of all players in squad
  • Each team plays each other twice (home + away) = 12 total matches
  • Win probability: 70% skill-based (strength difference) + 30% randomness
  • Generates full standings table with W/L/NRR/Points

Evaluation Output

Per Team: ├─ 10 grader scores (0.0-1.0 each) ├─ Composite score (weighted average) ├─ Rank (1st-4th) ├─ Best decision: "Bought sleeper X at ₹0.5Cr (true value ₹4.2Cr)" └─ Worst decision: "Overpaid ₹8Cr for trap Y (true value ₹1.1Cr)" Season Simulation: ┌──────┬────┬────┬─────┬────────┐ │ Team │ W │ L │ NRR │ Points │ ├──────┼────┼────┼─────┼────────┤ │ CSK │ 5 │ 1 │+0.8 │ 10 │ ← Winner │ MI │ 4 │ 2 │+0.3 │ 8 │ │ RCB │ 2 │ 4 │-0.4 │ 4 │ │ KKR │ 1 │ 5 │-0.7 │ 2 │ └──────┴────┴────┴─────┴────────┘
07 — External Agents

External Agent API

Any AI system can participate in an auction via the External Agent API. This allows independent agents (running in Docker, Claude Code, custom scripts, etc.) to compete against the built-in OpenRouter agents.

How It Works

1. SETUP Agent added with model = "external" → System generates a random 32-char hex token → Token displayed in lobby UI 2. CONNECT External agent polls: GET /external/state?token=TOKEN → First poll marks agent as "connected" → Green notification appears in lobby UI 3. PLAY During bidding loop, when it's the external agent's turn: │ ├─ Bidding loop sets pendingExternalBid in auction config ├─ UI shows "Waiting for External Agent" banner ├─ External agent sees your_turn: true in state response ├─ External agent POSTs: POST /external/bid?token=TOKEN │ Body: { action: "bid", amount: 2.4, reasoning: "..." } ├─ Bidding loop detects the bid in DB └─ Continues normally 4. TIMEOUT If external agent doesn't respond within 120 seconds → Auto-pass for that turn

09 — Prompt Deep Dive

Auction Prompt Structure

Each auction turn sends 3 message layers to the LLM:

1. System Prompt (rules book)
• Team identity ("You manage Mumbai Indians")
• 4 ranked objectives (squad > roles > quality > budget)
• Hard constraints (15-18 squad, 8 overseas, 100 Cr)
• All 10 scoring graders explained
• Player assessment framework per role
• Trap detection guidance
• Strategic phase advice (early/mid/late)
• Dynamic urgency warnings
2. Conversation History
Last 10 completed lots reconstructed as user/assistant pairs:

USER: LOT #8: Vikram Shah (BOWLER) — SOLD ₹3.2Cr
ASST: ACTION: bid, AMOUNT: 3.2, REASONING: Need bowlers...

Gives the LLM memory of its own past decisions.
3. User Prompt (decision card)
• Player: name, role → subType, age, nationality, base price, style tags
• Formatted batting line (avg, SR, matches, 100s, 50s)
• Formatted bowling line (economy, wickets, bowling avg)
• Recent form (last 5 matches summary)
• Current bid or "no bids yet"
• Team status: purse, reserve calculation, effective budget, squad size, overseas, role breakdown, needs
• Opponents: each rival's purse, squad size, overseas count
• Upcoming 15 players with key stats
• Auction progress + escalating urgency warnings

TourBench Prompt Structure

Each match prediction sends 2 messages:

System: Expert Analyst Persona
"You are an expert T20 cricket analyst. You understand how venue conditions (pace/spin, ground size, dew) interact with squad composition. Analyze squad strengths against venue traits, form, H2H, and standings."
User: Match Briefing
• Match number, type, team names
• Venue: description + pace/spin, batting conditions, boundaries, dew, pitch deterioration
• Squad composition per team: role counts, avg stats, pace vs spin bowlers, power hitters, top 5 players
• Points table (live standings)
• Head-to-head record between these teams
• Recent form (W/L string for last 3)
• All past match results
• 5-point analysis checklist to guide reasoning

TourBench Scoring Metrics (7 graders)

MetricWeightWhat It Measures
Accuracy25%Did you pick the correct winner? Simple win%.
Brier Score20%Probabilistic calibration — penalizes wrong high-confidence picks heavily. avg(1-prob)²
Upset Detection10%When the weaker team won (upset), did you predict it? Tests contrarian reasoning.
Margin Accuracy10%How close was "22 runs" to actual "18 runs"? Also: did you match margin type (runs vs wickets)?
Calibration15%Are high-confidence (≥0.7) predictions more accurate than low-confidence ones? Tests knowing what you know.
Consistency10%Correct + high confidence = good. Wrong + low confidence = also good. Rewards self-awareness.
Reasoning10%Placeholder for future LLM-graded reasoning quality (currently 0.5 baseline).
Key insight: Accuracy alone isn’t enough. A model that says 51% on every pick and gets 8/14 right scores lower than one that gets 7/14 but correctly assigns high confidence to its correct picks. Calibration rewards knowing what you know.

TourBench: Two Modes

Synthetic Mode
• 4 teams, 14 matches (12 league + qualifier + final)
• Random squad generation with deterministic simulation
• Fictional team aliases to prevent LLM knowledge leakage
• Tests pure analytical reasoning from provided data only
Real Data Mode (IPL Seasons)
• 4 seasons: S1 (59 matches), S2 (74), S3 (73), S4 (71)
• Real IPL team names, venues, match results
• Historical seasons visible for context; eval season hidden
• Key player rosters + head-to-head history from prior years
• Tests whether LLMs can leverage real cricket knowledge
